## Core Philosophy

> "The solution to city generation's complexity is not elimination, but organization."

Cities cannot be generated by noise alone — noise produces terrain, not place. The thesis rejects:
- Pure randomness / density-map approaches (no urban logic)
- Uniform grids (monotony, no character)
- L-systems (ignore real-world constraints)

The winning strategy marries **ArcGIS-style layered spatial data** with **game level design logic**:
- Each system is a layer (terrain → roads → parcels → zoning → buildings)
- Layers interact deterministically
- The editor gives authorial override at every level

## Algorithm 1 — Road Network Generation (Planar Graph Growth)

### Data Model
City roads = **planar graph** (no edge crossings in 2D, ignoring bridges/tunnels).
- **Node** = intersection or endpoint
- **Edge** = street segment
- **Face** = enclosed area between edges (= city block)
- **Degree** = number of edges at a node

### Phase 1 — Seed Node Placement

Two strategies:

**A. Nodes Around Seed Nodes ("Investment Capital" model)**
- Each seed node has a **Split Budget** (integer or float)
- Budget ≥ 1 → deterministically spawns children; 0 < budget < 1 → probabilistic
- Each child's budget = parent's budget − 1
- Chain terminates at budget = 0
- Multiple seed nodes → overlapping growth waves

**B. Nodes Inside a Loop (boundary fill)**
- Start with at least one point inside a closed boundary (e.g., a ring road)
- Repeatedly pick candidates at random radii/angles around existing nodes
- Accept candidate if: (a) inside boundary, (b) distance from all other nodes ≥ clearance
- Produces organic irregular fills within a region

### Phase 2 — Edge Creation

For each node (iterated until list is empty):
1. Skip if degree ≥ 4 (no 5-way intersections — unless explicitly wired as a junction)
2. Call **Create Street From Node**:
   - Pick a direction
   - Find the "best" candidate node within connection radius
   - Apply hard/soft constraints (below)
   - Link valid pair as a new Edge
   - Advance to new node; record direction; repeat

**Constraints (in evaluation order):**

| Constraint | Type | Notes |
|---|---|---|
| No crossing edges | Hard | Segment–segment intersection test; most expensive — optimize first |
| No duplicate edges | Hard | Checked before crossing (faster) |
| No triangles | Soft | Both nodes share a common neighbor → triangle → reject. Enforced automatically if min angle > 60° |
| Minimum angle | Soft | Dot product test; **55°–65° gives best results** |
| Max degree 4 | Soft | Optional; disable at designated junction nodes |

**Scoring (when multiple candidates pass constraints):**
1. Alignment with previous edge direction (prefer straight)
2. Lower distance (prefer nearby)

Isolated / unreachable nodes are removed after this phase.

### Phase 3 — Edge Splitting

Subdivide all edges into shorter uniform segments. New mid-points become nodes. This creates **anchor points** for secondary street connections in the next recursion.

### Phase 4 — Face Detection (City Blocks)

Algorithm: **rightmost-edge traversal** (chordless cycles in a planar graph)
1. Build directed edge set: every edge AB is stored as both A→B and B→A
2. While directed edges remain:
   - Pick any remaining directed edge to start
   - Traverse: at each node, take the **rightmost** available next edge
   - Continue until back at start → this encloses one **face**
   - Remove used directed edges; store face
3. The **outer face** (extends to infinity) needs special handling — use regular node generation for it

Each face is a city block. Store which face each edge separates (edge → (face_left, face_right) table) for neighbor lookups.

### Phase 5 — Recursive Subdivision

For every detected face (except outer), repeat Phase 1-B (nodes inside loop) at finer scale → connect with edges (Phase 2) → thinner streets are drawn. The "split number" controls recursion depth → more splits = more sprawl.

## Algorithm 2 — Zoning Assignment (Block Type Classification)

### Zone Types (canonical list from thesis)
1. City Center
2. Mixed Residential/Commercial
3. High Density Residential
4. Medium Density Residential
5. Low Density Residential
6. Commercial
7. Industrial
8. Business
9. Park

### Inputs Required
- **Adjacency Table**: NxN matrix, zone_i × zone_j → weight (positive = prefer adjacent, negative = avoid). Symmetric.
- **Target Proportions**: per-zone desired area fraction (sum = 1.0)
- **Distance Requirements**: per-zone function f(dist_from_center) → weight multiplier. Example: Industrial avoids center → f ramps 0→1 from 30%–50% radius.

### Algorithm (Greedy Flood-Fill)

```
remaining = all blocks except outer face
total_area = sum(remaining areas)
tally = {zone: 0.0 for zone in zones}
open = [center_block]   # or random seed

while remaining not empty:
    if open is not empty:
        block = pop(open)
    else:
        block = pop(remaining)

    type = PickType(block, tally, total_area)
    tally[type] += block.area
    RecalcGlobalWeights(tally, total_area)
    for neighbor in block.neighbors:
        if neighbor in remaining:
            move neighbor to open
```

### PickType(block)

For each neighboring edge:
- Get the zone of the block on the other side
- If assigned: add adjacency_table[neighbor_zone] × edge_length to running weight vector

Then for each zone:
```
weight[zone] *= dist_requirement_fn(block.dist_from_center)
weight[zone] *= global_weight[zone]
```

Normalize and sample.

### Global Weights (dynamic rebalancing)

```
global_weight[zone] = (remaining_zone_area%) / (remaining_total_area%)
```

If a zone is underrepresented → weight > 1 (boosted). Overrepresented → weight < 1 (suppressed). This steers the final distribution toward target proportions without hard enforcement.

## Key Parameters Per RC Generator Pass

| Parameter | Role |
|---|---|
| `seed` | Random seed for node placement |
| `starting_graph` | Seed nodes / initial roads (axiom input) |
| `split_numbers` | Budget per seed node (controls spread) |
| `min_angle` | Edge constraint; 55°–65° recommended |
| `clearance` | Min distance between nodes per recursion level |
| `extension_range` | [min, max] distance for child nodes |
| `connection_radius` | Max edge length (≈ extension_range.max works well) |
| `edge_split_distance` | Segment length before face detection |

## Design Intent Summary (quote from thesis)

> "The task here is to bridge industrial-grade spatial thinking (ArcGIS and friends) with the intuitive, legible logic of game systems — so the result is not just a map, but a place."

Generators must produce cities that are:
- **Structurally believable** (planar graph, hierarchical road network)
- **Functionally correct** (zoning logic, adjacency constraints)
- **Authorially controllable** (every parameter is a designer knob)
- **Deterministic** (same seed + axiom = same city)

## Further Improvements Noted in Thesis (not yet implemented)

- Classify faces between recursion levels → apply different generation settings per face class (e.g., denser roads near city center face vs. suburban face)
- Terrain constraints: exclude zones using pre-placed boundary edges; no-crossing rule automatically routes around them
- Bridges/tunnels: take a degree-4 node and reconnect opposite-pair neighbors
- Planned grid cities: generate nodes on a grid with high minimum angle (≥60°)
- Incorporate terrain data (slope, elevation, proximity to water) into node generation or scoring functions
