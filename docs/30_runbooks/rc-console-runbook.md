# rc-console Runbook

`rc-console` is the interactive, terminal-based Read-Eval-Print Loop (REPL) for the `RogueCityVisualizerHeadless` application. It bridges the gap between the dev-shell environment and the live, running visualizer state, allowing developers and AI agents to manipulate the UI and data layers asynchronously.

## Launching the Console
From the standard `dev-shell.ps1`, you can launch the console by simply typing:
```powershell
rc-console
```

When launched, you will see a  ASCII banner and a `rc> ` prompt. The headless engine loop runs continuously in the background at 60 FPS, ensuring that state changes and subprocesses don't block the actual simulation context.

## Subprocess Execution Commands
The console comes equipped with a `_popen` driven subprocess engine that can run native `dev-shell` commands and standard Windows PowerShell pipelines under the hood. Their outputs are streamed back to the console in `DarkGray` formatting.

- `! <cmd>` or `shell <cmd>`: Executes an arbitrary shell command (e.g., `! echo Hello World`).
- `build`: Invokes `rc-bld-headless` internally to recompile the visualization targets.
- `doctor`: Runs `rc-doctor` environment diagnostics.
- `problems`: Fetches and prints the latest compilation output via `rc-problems`.
- `ai_start`: Spins up the local AI bridge server via `rc-ai-start-wsl` without needing a new terminal tab.
- `ai_query <prompt>`: Queries the local model architecture seamlessly with `rc-ai-query`.

## UI & Application State Commands
These commands directly interface with the `GlobalState` and ImGui contexts of the running headless visualization.

- `dump_ui`: Dumps the entire ImGui UI Introspection tree into a formatted JSON string. This is exceptionally useful for AI agents trying to "see" the active panel layouts.
- `state`: Queries domain information from `GlobalState`, providing quick feedback on elements like road and district counts.
- `rc-log` or `log`: Opens the interactive TUI Dashboard Viewer. The dashboard displays logs generated by the application, separated into tabs (`BUILD`, `TUI`, `GUI`, `AI`).
  - Use **[Tab]** to cycle through the available tabs.
  - Use **[Backspace]** to exit the dashboard and return to the main REPL prompt.
  - Logs are concurrently persisted as JSON Lines (`visualizer.jsonl`) for backend telemetry.
- `hfsm <event>`: Directly fires `EditorEvent` state changes to simulate user interaction (e.g., `hfsm Tool_Roads` or `hfsm GotoIdle`).
- `pause`: Pauses the continuous frame generation loop.
- `resume` / `play`: Unpauses the frame generation loop.
- `step [N]`: Steps the frame loop manually for `N` frames.
- `exit` or `quit`: Safely breaks the loop and triggers a graceful shutdown and thread-join process.

## Advanced AI Agent usage
Because the console runs asynchronously and uses `std::flush` for all outputs, it is heavily optimized for zero-latency piped communication. An AI agent script can easily invoke the headless visualizer with `--interactive` and pipe commands directly into it to retrieve JSON UI states (`dump_ui`) or test `hfsm` state machines without maintaining heavy graphical infrastructure.
